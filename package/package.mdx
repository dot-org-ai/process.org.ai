# process.org.ai

APQC Process Classification Framework with 1,921 processes for Business-as-Code

## Installation

```bash
npm install process.org.ai
# or
pnpm add process.org.ai
# or
yarn add process.org.ai
```

## Usage

```typescript
import {{ $ }} from 'sdk.do'
import type {{ Process }} from 'process.org.ai'

// Semantic patterns
const item = await $.Process.create({{
  $type: 'Process',
  name: 'Example'
}})

const result = await $.Process.get('item-id')
```

## Tree-Shakable Exports

```typescript
// Import specific types
import {{ Process }} from 'process.org.ai/types'

// Import specific data
import {{ ProcessData }} from 'process.org.ai/data'

// Import Zod schemas
import {{ ProcessSchema }} from 'process.org.ai/schemas'
```

## Build Process

This package is built using **mdxe** (MDX Execution Engine) with zero configuration:

1. **Parse MDX** - All `.mdx` files in `things/` directory
2. **Extract Frontmatter** - YAML frontmatter becomes structured data
3. **Generate Types** - TypeScript interfaces from frontmatter
4. **Generate Schemas** - Zod schemas for validation
5. **Generate Data** - Tree-shakable metadata exports
6. **Build Hierarchy** - Parent-child relationships from `subClassOf`
7. **Compile** - ESM + CJS bundles with declarations

No config files needed - everything is derived from MDX frontmatter with `$type: https://schema.org.ai/Package`.

## Features

- **2,704 MDX files** (1.41x expansion from "and"/"or" splitting)
- GraphDL semantic naming (verb.Object.preposition.Object)
- 5-level hierarchy (Category → Process Group → Process → Activity → Task)
- Parent-child navigation
- Business-as-Code integration
- APQC code cross-reference
